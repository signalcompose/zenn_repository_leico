[前回のチャプター](./23) までで、実践的な C++ によるBlueprintノード作成を一通り行いました。
様々な内容を扱ってきたため、一度振り返りをしておきます。


## `StartSequence` / `StopSequence` / `Sequence` ノード

[Chapter 19](./19) 、 [Chapter 20](./20) 、 [Chapter 21](./21) では、MIDI ファイルの再生処理と、
それに必要な MIDI イベント用クラスの作成を行いました。
これらのノードは **インスタンスメンバー関数** として実装しており、使用時にはインスタンスを入力として渡す必要があります。  
逆に言えば、入力インスタンスを切り替えることで、複数の MIDI ファイルを個別に制御できます。

## `BranchMidiEvent` ノード

[Chapter 22](./22) では、MIDI イベントの種類（ノートオン／オフ、CCなど）に応じて実行ピンを分岐させる
`BranchMidiEvent` ノードを作成しました。

このノードの実装を通じて、**複数の実行ピンを持つ Blueprint ノード** の作成方法を学ぶことができます。

## `ParseMidiEvent` / `ParseCC` ノード

[Chapter 23](./23) では、MIDI イベントから情報を抽出して出力する以下のノードを作成しました:

- `ParseMidiEvent` : ノート情報(チャンネル、ノートナンバー、ベロシティ)を出力
- `ParseCC` : コントロールチェンジ情報(チャンネル、番号、設定値)を出力

これらノードの実装を通じて、**複数のデータを出力する Blueprint ノード** の作成方法を学ぶことができます。

## `CreateMidiFileSequence` ノード

[Chapter 18](./18) では、読み込んだ MIDI ファイルアセットから再生用インスタンスを生成する  
`CreateMidiFileSequence` ノードを実装しました。

このノードの実装を通じては、**エディタ拡張で読み込んだファイルを Blueprint 上で利用する** 方法を学ぶことができます。

## Blueprint まとめ

ここまでのチャプターでは、以下のような Blueprint ノードの作成方法を学びました:

- インスタンスメンバーとなるノード
- 複数の実行ピンを持つノード
- 複数データを出力するノード
- 読み込み機能を自作したアセットを利用する方法

これらの知識・経験は、Unreal Engine 上で Blueprint を活用する際や、
独自の Blueprint ノードを含むプラグインを開発する際に役立つことと思います。
